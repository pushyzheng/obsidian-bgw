#JVM 

## 你有哪些手段来排查  JVM 的问题?

1. jmap
	- 观察JVM **各个区域的运行情况**
	- 内存占用
	- dump 堆存储分析
2. jstat
	- 查看 GC 的情况，例如 **fullgc 次数**等
3. jstack
	- 线程运行情况（状态、调用栈），排查**阻塞、死锁**情况
4. 火焰图
	- 用来分析**系统性能**问题，某段代码的执行耗时太长等

## OOM 可能会有哪些原因？

1. 请求创建一个超大对象（大数组），未被正常释放
2. 超出预期的访问量、数据量（上游系统流量突增）
3. 内存泄漏
